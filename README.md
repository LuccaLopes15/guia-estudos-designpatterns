# Guia de Estudos: Design Patterns com NotebookLM 🧠🚀

Este repositório serve como um guia de estudos centralizado e documentação para o aprendizado de **Design Patterns (Padrões de Projetos)**. O projeto foi estruturado a partir da criação de um **NotebookLM** (ferramenta de IA do Google para gerenciamento e síntese de conhecimento), alimentado por fontes de alta qualidade no cenário de desenvolvimento de software orientado a objetos.

Aqui você encontrará a organização dos conceitos, objetivos de estudo e o mapeamento dos padrões explorados ao longo dessa jornada técnica.

---

## 🎯 Objetivo do Estudo

O principal objetivo deste guia é dominar os conceitos fundamentais e avançados de Padrões de Projetos, capacitando a tomada de decisões arquiteturais mais maduras. Os focos específicos são:

* **Compreensão Teórica:** Entender o problema real que cada padrão resolve, sua estrutura (classes, interfaces e interações) e as trade-offs (vantagens e desvantagens).
* **Aplicação Prática:** Identificar cenários no desenvolvimento de software do dia a dia onde a aplicação de um padrão resulta em código mais limpo, manutenível e extensível.
* **Princípios S.O.L.I.D. & POO:** Reforçar a base em Programação Orientada a Objetos, conectando diretamente os padrões aos princípios de design de código.
* **Construção de Repertório:** Desenvolver um vocabulário técnico comum compartilhado por equipes de engenharia de software de alto nível.

## 📚 Fontes de Conhecimento Utilizadas

O **NotebookLM** deste projeto foi alimentado utilizando três pilares fundamentais, combinando teoria clássica, exemplos visuais interativos e explicações práticas em vídeo:

1.  **[Livro Clássico] Padrões de Projetos: Soluções Reutilizáveis de Software Orientados a Objetos**
    * *Autores:* Erich Gamma, Richard Helm, Ralph Johnson, John Vlissides (conhecidos consensualmente como *Gang of Four - GoF*).
    * *Papel no Estudo:* Base conceitual acadêmica, detalhamento formal das estruturas e o catálogo original dos 23 padrões fundamentais.
2.  **[Plataforma Web] Refactoring.Guru**
    * *Website:* [refactoring.guru](https://refactoring.guru)
    * *Papel no Estudo:* Abordagem visual excelente, diagramas claros, pseudocódigo intuitivo e analogias do mundo real que facilitam a fixação de como cada padrão opera.
3.  **[Playlist de Vídeos] Design Patterns - Renato Augusto**
    * *Link da Playlist:* [YouTube - Design Patterns (Renato Augusto)](https://www.youtube.com/playlist?list=PLNHxHgB-_LTt67szNmMsZwqBKq9jH4uKJ)
    * *Papel no Estudo:* Explicação dinâmica, implementações práticas, discussões em português com exemplos voltados para o mercado de desenvolvimento moderno.
